Authentication

Valeria provides a straightforward way to handle user authentication including login, logout, and session management. It supports middleware for route protection and integrates seamlessly with JWT.

Login Example

The login controller verifies user credentials and returns a JWT token or starts a session: 

<?php
namespace App\Controllers;

use Valeria\Http\Request;
use Valeria\Http\Response;
use Valeria\Auth\JWT;
use App\Models\User;

class AuthController
{
    public function login(Request $request, Response $response)
    {
        $email = $request->input('email');
        $password = $request->input('password');

        $user = User::where('email', $email)->first();

        if (!$user || !password_verify($password, $user->password)) {
            return $response->json(['error' => 'Invalid credentials'], 401);
        }

        $payload = [
            'sub' => $user->id,
            'email' => $user->email,
            'iat' => time(),
            'exp' => time() + 3600,
        ];

        $token = JWT::encode($payload, getenv('JWT_SECRET'));

        return $response->json(['token' => $token]);
    }

    public function logout(Request $request, Response $response)
    {
        // For JWT, logout can be handled on client by deleting token
        return $response->json(['message' => 'Logged out successfully']);
    }
}

Protecting Routes with Middleware

Use the #[Middleware('auth')] attribute to secure controllers or routes: 

#[Middleware('auth')]
public function dashboard()
{
    // This method requires the user to be authenticated
}

Session Management

Alternatively, Valeria can manage sessions for traditional stateful authentication: 

<?php
session_start();

if ($authenticated) {
    $_SESSION['user_id'] = $user->id;
} else {
    session_destroy();
}
Tip: Combine JWT and Middleware features for secure and scalable API authentication.
