Middleware

Middleware allows you to filter HTTP requests entering your application. It acts as a layer that can intercept requests before they reach your controllers. Use middleware to implement authentication, logging, CORS, or any pre/post request logic.

Defining Middleware

Middleware classes must implement a handle method which receives the request and a next closure to forward the request. 

<?php
namespace App\Middleware;

class AuthMiddleware
{
    public function handle($request, $next)
    {
        if (!isset($_SESSION['user_id'])) {
            header('Location: /login');
            exit;
        }

        return $next($request);
    }
}

Registering Middleware

Middleware are registered in the core framework configuration and can be assigned globally or to specific routes.

Assign Middleware to Routes

Use the #[Middleware('auth')] attribute above your controller method to apply middleware: 

#[Middleware('auth')]
public function dashboard()
{
    // This route requires the user to be authenticated
}

Middleware Pipeline

When a request is dispatched, Valeria builds a middleware pipeline and executes each middleware in sequence. The request passes through each middleware until it reaches the controller.

Note: Middleware provide a clean and reusable way to handle common logic and keep your controllers focused on business logic.
