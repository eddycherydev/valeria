Configuration

Valeria uses environment variables to manage configuration settings securely and flexibly. These variables are loaded from a .env file located in the project root.

Example .env file

APP_NAME=Valeria
APP_ENV=development
APP_DEBUG=true
APP_URL=http://localhost:8000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=valeria
DB_USERNAME=root
DB_PASSWORD=

JWT_SECRET=your_secret_key_here

Loading Environment Variables

Valeria includes a simple Env class to load and access environment variables: 

<?php
namespace Core;

class Env
{
    protected static array $variables = [];

    public static function load(string $path): void
    {
        if (!file_exists($path)) {
            throw new \Exception(".env file not found at $path");
        }

        $lines = file($path, FILE_IGNORE_NEW_LINES | FILE_SKIP_EMPTY_LINES);

        foreach ($lines as $line) {
            if (str_starts_with($line, '#')) {
                continue;
            }

            [$key, $value] = explode('=', $line, 2);
            self::$variables[trim($key)] = trim($value);
        }
    }

    public static function get(string $key, $default = null)
    {
        return self::$variables[$key] ?? $default;
    }
}

Accessing Configurations

Use Env::get() anywhere in your application to retrieve configuration values: 

<?php
use Core\Env;

Env::load(__DIR__ . '/../.env');

$dbHost = Env::get('DB_HOST', '127.0.0.1');
Tip: Keep sensitive information like passwords and API keys in the .env file and never commit it to version control.
