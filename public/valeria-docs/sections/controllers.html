Controllers

Controllers are classes responsible for handling the incoming HTTP requests and returning responses. They are located in the app/Controllers/ directory.

Creating a Controller

A controller is just a PHP class with public methods that act as route handlers. 

<?php
namespace App\Controllers;

use Core\Request;
use Core\Response;

class HomeController
{
    public function index(Request $request): Response
    {
        return new Response(['message' => 'Welcome to Valeria']);
    }

    public function about(): Response
    {
        return new Response(['app' => 'Valeria', 'version' => '1.0.0']);
    }
}

Registering Controller Routes

Use the route definitions in config/routes.php to point to controller actions. 

$router->get('/', [HomeController::class, 'index']);
$router->get('/about', [HomeController::class, 'about']);

Dependency Injection

You can type-hint request objects or services in controller methods, and Valeria will automatically inject them.

Using Attributes (e.g. Middleware)

Controllers support PHP 8+ attributes. For example, you can use #[Middleware('auth')] to protect routes: 

<?php
namespace App\Controllers;

use Core\Attributes\Middleware;

class DashboardController
{
    #[Middleware('auth')]
    public function index()
    {
        return 'Welcome, authenticated user!';
    }
}
